Klone Operator Helm Repository

Welcome to the official Helm repository for Klone Operator - a Kubernetes operator that enables you to deploy nested K3s clusters inside an existing Kubernetes cluster.

Quick Start

Add the Helm repository:

helm repo add klone https://raghavendiran-2002.github.io/klone/helm/packages/
helm repo update

Install the Klone Operator:

helm install klone-operator klone/klone-operator \
  --namespace klone \
  --create-namespace

What is Klone?

Klone is a Kubernetes Operator that allows you to:

  • Deploy nested K3s clusters with configurable control plane and worker nodes
  • Automatic CIDR allocation to prevent network conflicts
  • Web terminal access via ttyd with automatic kubeconfig provisioning
  • Multiple ingress options: Tailscale, AWS ALB, or none
  • Optional metrics-server auto-installation in nested clusters
Note: Each KloneCluster creates an isolated k3s cluster with its own control plane, workers, and optional web terminal access.

Create Your First Cluster

After installing the operator, create a KloneCluster resource:

kubectl apply -f - <<EOF
apiVersion: klone.klone.io/v1alpha1
kind: KloneCluster
metadata:
  name: my-cluster
spec:
  controlPlane:
    replicas: 1
  workers:
    replicas: 2
  terminal:
    enabled: true
  ingress:
    type: none
EOF

Configuration

Customize the installation with your own values.yaml:

controllerManager:
  image:
    tag: v1.0.51
  resources:
    limits:
      cpu: 1000m
      memory: 256Mi

Install with custom values:

helm install klone-operator klone/klone-operator \
  -f values.yaml \
  --namespace klone \
  --create-namespace

  • Cluster admin permissions

Verification

Check the operator is running:

kubectl get pods -n klone

View operator logs:

kubectl logs -n klone -l control-plane=controller-manager -f

Check your KloneCluster status:

kubectl get klonecluster my-cluster -o yaml
